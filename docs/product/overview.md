# 产品概览

Zenith Admin 是一个面向企业后台、运营平台和内部工具场景的全栈后台管理系统。

## 核心定位

它的定位不是“做一个空白管理台壳子”，而是提供一套已经具备基础管理能力、并且适合继续扩展的项目底座。

适合这些团队：

- 想快速上线后台系统的创业团队
- 需要统一技术栈和代码规范的中小团队
- 需要做客户演示、PoC 或内部运营工具的项目组

## 技术栈

| 层级 | 技术 |
| --- | --- |
| 后端 | Hono v4 + Node.js |
| 前端 | React 19 + Vite 6 |
| UI | Semi Design v2 |
| ORM | Drizzle ORM |
| 数据库 | PostgreSQL |
| 缓存/会话 | Redis（ioredis） |
| 校验 | Zod |
| 认证 | JWT Bearer Token（双 Token） |

## 产品价值

### 降低初始化成本

用户、角色、菜单、字典、文件管理等后台常见模块已经具备基础能力，不必每个项目都重头搭建。

### 内置工作流引擎

可视化流程设计器（审批/抄送/条件分支）、表单字段设计、流程自动化与事件总线，无需引入外部引擎即可覆盖大多数审批场景。

### 即时通讯开箱可用

基于 WebSocket 的单聊/群聊、消息回复/撤回/转发、@提及、投票等能力，快捷聊天可在任意页面唤起。

### 多渠道通知体系

站内信、公告推送、邮件（SMTP）、短信（多服务商）均已落地，并配备模板管理与发送日志。

### 安全能力全链路覆盖

JWT 双 Token + 会话黑名单 + 登录失败锁定 + IP 黑白名单 + 数据脱敏 + 幂等防重复提交 + 接口限流，满足企业级后台安全合规要求。

### AI 能力预埋

内置多服务商 AI 对话（支持 PDF 文档解析）与 AI 服务商管理页面，用户可按需配置自己的模型密钥；工程结构配合 Zenith Skill 可通过 AI 指令快速生成 CRUD 模块。

### 保持演进空间

`shared` 层统一类型、常量和 schema，适合在新增业务时保持前后端一致性。

### 面向团队协作

接口规范、分页格式、页面布局、时间处理方式等都已经形成约定，适合多人协作开发。
